1. 개요
-------
본 매뉴얼은 보안사고 발생 시 현장 담당자가 즉시 참고할 수 있는
실무 대응 절차를 상세히 안내합니다.

3. 사고 등급별 대응 절차
-----------------------

3.1 1급 사고 (최중대)
─────────────────────
정의: 대규모 고객정보 유출, 금융거래 중단, 랜섬웨어 전사 감염

【즉시 조치 - 5분 이내】
□ 보안관제센터 연락 (전화 필수)
□ 피해 시스템 네트워크 격리 (케이블 분리 또는 포트 차단)
□ 현장 보존 (시스템 끄지 않기, 로그 확보)

【30분 이내 조치】
□ CISO 보고
□ 대표이사 보고 (CISO → 대표)
□ 비상대책본부 소집 검토
□ 현장 출입 통제

【1시간 이내 조치】
□ 금융당국 초기 보고 준비
□ 외부 포렌식 업체 연락 검토
□ 임원진 상황 브리핑

【4시간 이내 조치】
□ 피해 범위 파악
□ 복구 계획 수립
□ 대외 커뮤니케이션 준비 (홍보팀)

3.2 2급 사고 (중대)
───────────────────
정의: 내부정보 유출, 다수 시스템 악성코드 감염, 핵심시스템 침해시도

【30분 이내 조치】
□ 보안관제센터 연락
□ 피해 시스템 격리 검토
□ CISO 보고

【2시간 이내 조치】
□ 원인 분석 착수
□ 추가 피해 방지 조치
□ 관련 부서 통보

【4시간 이내 조치】
□ 상세 분석 및 대응
□ 복구 작업 (안전 확인 후)

3.3 3급 사고 (보통)
───────────────────
정의: 일부 악성코드 감염, 정책 위반, 접근권한 오남용

【4시간 이내 조치】
□ 보안팀 보고
□ 담당자 조치
□ 원인 분석

【당일 조치】
□ 시스템 정상화
□ 재발 방지 조치
□ 보고서 작성

3.4 4급 사고 (경미)
───────────────────
정의: 단순 침해시도, 경미한 위반

【익영업일 조치】
□ 보안팀 보고
□ 담당자 확인
□ 필요시 조치

4. 사고 유형별 대응
------------------

4.1 랜섬웨어 감염
─────────────────
【발견 즉시】
⚠ 절대 금지: 랜섬 지불, 파일 열기 시도, 시스템 재부팅

✓ 해야 할 일:
① 네트워크 케이블 즉시 분리 (확산 방지)
② 전원 유지 (끄지 않음 - 증거 보존)
③ 보안관제센터 즉시 연락
④ 화면 사진 촬영 (랜섬 메시지)
⑤ 감염 시점, 수상한 행위 메모

【보안팀 도착 후】
- 메모리 덤프 확보
- 디스크 이미징
- 감염 경로 추적
- 백업 상태 확인

4.2 악성코드 감염
─────────────────
【발견 즉시】
① 네트워크 분리
② 보안팀 신고
③ 백신 검사 실행 (삭제하지 않음 - 분석용)

【보안팀 조치】
- 악성코드 샘플 확보
- 전사 추가 감염 확인
- 유포 경로 차단

4.3 해킹/침입 탐지
─────────────────
【발견 즉시】
① 계정 비밀번호 변경 금지 (증거 훼손 방지)
② 보안관제센터 즉시 연락
③ 로그인 세션 유지

【보안팀 조치】
- 접근 로그 분석
- 침입 경로 추적
- 권한 변경 확인
- 데이터 유출 확인

4.4 개인정보 유출
─────────────────
【발견 즉시】
① 유출 경로 차단 (가능한 경우)
② 보안팀 즉시 보고
③ 유출 범위 파악

【필수 조치】
- 개인정보보호위원회 신고 (1천명 이상: 72시간 내)
- 정보주체 통지
- 원인 분석
- 재발 방지

4.5 DDoS 공격
─────────────
【발견 즉시】
① 네트워크팀 즉시 연락
② 트래픽 모니터링
③ 공격 IP 확인

【대응 조치】
- ISP 협조 요청 (트래픽 클리닝)
- 방화벽 차단 룰 적용
- CDN/Anti-DDoS 서비스 활성화

5. 증거 보존 절차
----------------

5.1 증거 보존 원칙
- 원본 시스템 손대지 않기
- 모든 조치 시간/내용 기록
- 사진/영상 촬영
- Chain of Custody 유지

5.2 증거 수집 순서
① 휘발성 데이터 (먼저)
   - 메모리 덤프
   - 네트워크 연결 상태
   - 실행 중인 프로세스
   - 로그인 세션

② 비휘발성 데이터
   - 시스템 로그
   - 이벤트 로그
   - 디스크 이미지

5.3 증거 수집 도구
- 메모리 덤프: FTK Imager, DumpIt
- 디스크 이미징: FTK Imager, dd
- 해시값: md5sum, sha256sum

5.4 증거 보관
- 원본 격리 보관
- 복사본으로 분석
- 해시값 기록
- 접근 이력 관리

================================================================================

6. 복구 절차
-----------

6.1 복구 전 확인
□ 원인 분석 완료
□ 악성코드 완전 제거 확인
□ 취약점 패치 완료
□ 백업 데이터 무결성 확인

6.2 복구 순서
① 핵심 시스템 우선
② 백업으로부터 복원
③ 보안 패치 적용
④ 비밀번호 전면 변경
⑤ 정상 동작 확인
⑥ 모니터링 강화

6.3 서비스 재개 승인
- 3급 이하: 보안팀장 승인
- 2급: CISO 승인
- 1급: 대표이사 승인

================================================================================

7. 보고서 작성
-------------

7.1 초기 보고서 (사고 후 24시간 내)
- 사고 개요
- 발견 경위
- 현재 상황
- 즉시 조치 사항
- 예상 영향

7.2 상세 보고서 (사고 종료 후 5영업일 내)
- 사고 타임라인
- 근본 원인 분석
- 피해 현황
- 대응 조치 내역
- 재발 방지 대책
- 교훈 (Lessons Learned)

7.3 보고 양식
- 사내 포털 > 보안 > 사고보고서 양식 다운로드
- 또는 보안팀에 요청

================================================================================
