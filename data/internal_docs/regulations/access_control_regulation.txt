                              접근통제규정
================================================================================

제1장 총칙

제1조 (목적)
본 규정은 정보보안정책에 따라 정보시스템 및 데이터에 대한 접근을 통제하고
비인가 접근을 방지하기 위한 세부사항을 정함을 목적으로 한다.

제2조 (적용범위)
① 본 규정은 당사의 모든 정보시스템에 적용된다.
② 정보시스템이란 서버, 네트워크, 애플리케이션, 데이터베이스, 클라우드 서비스를 포함한다.
③ 임직원, 협력업체, 외부 접속자 등 시스템에 접근하는 모든 자에게 적용된다.

제3조 (용어의 정의)
① "사용자 ID"란 시스템 사용자를 고유하게 식별하는 계정을 말한다.
② "인증"이란 사용자가 주장하는 신원을 확인하는 과정을 말한다.
③ "인가"란 인증된 사용자에게 특정 자원에 대한 접근을 허용하는 것을 말한다.
④ "특권계정"이란 시스템 관리자 권한을 가진 계정을 말한다.
⑤ "2팩터 인증"이란 서로 다른 2가지 인증수단을 조합한 인증을 말한다.

--------------------------------------------------------------------------------

제2장 접근통제 원칙

제4조 (최소권한 원칙)
① 모든 사용자는 업무 수행에 필요한 최소한의 권한만 부여받는다.
② 불필요한 권한은 즉시 회수한다.
③ 권한 부여 시 업무 필요성을 소명하여야 한다.

제5조 (직무분리 원칙)
① 상호 견제가 필요한 직무는 분리하여 운영한다.
② 분리 필요 직무:
   - 개발 / 운영
   - 요청 / 승인
   - 실행 / 감사
③ 불가피하게 겸직 시 보완통제 적용 (로그 강화, 상시 모니터링)

제6조 (알 필요성 원칙)
① 정보 접근은 업무상 알아야 할 필요가 있는 자에게만 허용한다.
② 호기심에 의한 접근, 업무 범위 외 접근은 금지한다.

--------------------------------------------------------------------------------

제3장 사용자 계정 관리

제7조 (계정 생성)
① 계정 생성 절차:
   1. 소속 부서장이 접근권한 신청서 작성
   2. 정보보안담당자 검토
   3. 시스템 관리자 계정 생성
   4. 신청자에게 계정 정보 전달 (초기 비밀번호 별도)

② 계정 명명규칙:
   - 일반 계정: 사번 기반 (예: EMP12345)
   - 협력업체: 업체코드+순번 (예: VND001_01)
   - 시스템 계정: 용도+시스템명 (예: SVC_BATCH_ERP)

③ 계정 생성 시 필수 정보:
   - 사용자 성명, 부서, 직급
   - 접근 필요 시스템 목록
   - 필요 권한 수준
   - 사용 기간 (협력업체는 프로젝트 종료일)

제8조 (계정 유형)
| 유형 | 설명 | 관리 기준 |
|------|------|-----------|
| 일반계정 | 임직원 업무용 | 인사 연동, 퇴직 시 자동 정지 |
| 협력업체계정 | 외부 인력용 | 기간 만료 자동 정지, 연장 시 재승인 |
| 시스템계정 | 배치, 연동용 | 담당자 지정, 비밀번호 금고 보관 |
| 비상계정 | 장애 복구용 | 봉인 보관, 사용 시 즉시 보고 |
| 특권계정 | 관리자 권한 | 개인별 분리, 특별 관리 |

제9조 (계정 변경)
① 인사발령(전배, 승진) 시 72시간 내 권한 재검토
② 권한 변경 절차:
   1. 소속 부서장 변경 신청
   2. 기존 권한 회수 확인
   3. 신규 권한 부여

제10조 (계정 정지 및 삭제)
① 즉시 정지 사유:
   - 퇴직, 휴직
   - 보안사고 관련자
   - 장기 미사용 (90일 이상)
   - 비밀번호 5회 연속 오류

② 계정 삭제:
   - 퇴직 후 30일
   - 협력업체: 프로젝트 종료 후 7일
   - 삭제 전 로그 보관 확인

③ 계정 정지 해제:
   - 본인 확인 후 정보보안담당자 승인
   - 비밀번호 재설정 필수

--------------------------------------------------------------------------------

제4장 비밀번호 관리

제11조 (비밀번호 정책)
① 비밀번호 복잡도:
   - 최소 12자 이상
   - 영문 대문자, 소문자, 숫자, 특수문자 중 3종 이상 조합
   - 사전 단어, 개인정보 포함 금지
   - 연속 문자/숫자 3자 이상 금지 (예: abc, 123)

② 비밀번호 변경:
   - 일반계정: 90일마다 변경
   - 특권계정: 30일마다 변경
   - 의심 상황 시 즉시 변경

③ 비밀번호 이력:
   - 직전 5개 비밀번호 재사용 금지

제12조 (비밀번호 보호)
① 비밀번호는 암호화하여 저장 (SHA-256 이상)
② 비밀번호 공유 금지 (타인에게 알려주기 금지)
③ 비밀번호 메모 금지 (포스트잇, 파일 저장 금지)
④ 초기 비밀번호는 최초 로그인 시 즉시 변경

제13조 (비밀번호 분실)
① 분실 신고: IT헬프데스크 또는 셀프서비스 포털
② 본인 확인 후 임시 비밀번호 발급
③ 임시 비밀번호는 24시간 내 변경 필수
④ 본인 확인 방법: 사번 + 등록 휴대폰 인증 또는 대면 확인

--------------------------------------------------------------------------------

제5장 다중 인증

제14조 (2팩터 인증 적용 대상)
다음의 경우 2팩터 인증을 필수 적용한다:
① 금융거래시스템 접근
② 고객정보시스템 접근
③ 특권계정 사용
④ 외부망에서 내부 접근 (VPN)
⑤ 클라우드 관리 콘솔 접근
⑥ 원격 서버 접속

제15조 (인증 수단)
① 지원 인증 수단:
   - OTP (Time-based One-Time Password)
   - SMS/이메일 인증코드
   - 모바일 인증 앱 (회사 제공)
   - 생체인증 (지문, 안면)
   - 물리적 보안키 (FIDO2)

② 인증 수단 조합:
   - 1팩터: ID + 비밀번호
   - 2팩터: ID + 비밀번호 + OTP/생체인증

③ 인증 수단 분실:
   - 즉시 IT헬프데스크 신고
   - 임시 인증 수단 발급 (24시간 유효)
   - 신규 인증 수단 재등록

--------------------------------------------------------------------------------

제6장 접근권한 관리

제16조 (권한 신청)
① 신청 절차:
   1. 사용자: 접근권한 신청서 작성 (시스템, 권한 수준, 사유)
   2. 소속 부서장: 업무 필요성 확인 및 승인
   3. 데이터 소유부서: 데이터 접근 시 추가 승인
   4. 정보보안담당자: 보안 검토
   5. 시스템 관리자: 권한 부여

② 승인권한:
| 권한 수준 | 승인권자 |
|-----------|----------|
| 조회 권한 | 팀장 |
| 수정 권한 | 부서장 |
| 삭제 권한 | 본부장 + 정보보안담당자 |
| 관리자 권한 | CISO |

제17조 (권한 유형)
| 권한 | 설명 | 부여 기준 |
|------|------|-----------|
| 읽기(R) | 조회만 가능 | 업무 참고 필요 시 |
| 쓰기(W) | 생성, 수정 가능 | 데이터 처리 업무 |
| 삭제(D) | 삭제 가능 | 데이터 관리 책임 |
| 실행(X) | 프로그램 실행 | 특정 기능 수행 |
| 관리(A) | 권한 부여/회수 | 시스템 관리자 |

제18조 (권한 검토)
① 정기 검토:
   - 분기 1회 부서별 권한 목록 검토
   - 연 1회 전사 권한 일제 검토

② 검토 항목:
   - 퇴직/전배자 잔존 권한
   - 업무 불필요 권한
   - 과다 권한 (최소권한 원칙 위배)
   - 미사용 권한 (90일 이상)

③ 검토 결과 조치:
   - 불필요 권한 즉시 회수
   - 예외 사항은 사유 기록 및 상위 결재

--------------------------------------------------------------------------------

제7장 특권계정 관리

제19조 (특권계정 정의)
특권계정이란 다음의 권한을 가진 계정을 말한다:
① 시스템 관리자 (root, administrator)
② 데이터베이스 관리자 (DBA)
③ 네트워크 관리자
④ 보안 솔루션 관리자
⑤ 애플리케이션 관리자 (Admin 권한)

제20조 (특권계정 관리 원칙)
① 개인별 분리: 공용 특권계정 사용 금지, 개인별 관리자 계정 발급
② 사용 최소화: 관리 작업 시에만 사용, 일상 업무는 일반계정
③ 사용 승인: 특권계정 사용 시 사전 승인 (긴급 시 사후 보고)
④ 로그 기록: 모든 작업 상세 로깅
⑤ 정기 검토: 월 1회 특권계정 목록 및 사용 현황 검토

제21조 (특권계정 사용 절차)
① 사전 승인 (일반):
   1. 사용 신청 (작업 내용, 예상 시간)
   2. 팀장 승인
   3. 특권계정 체크아웃 (PAM 시스템)
   4. 작업 수행 (세션 녹화)
   5. 작업 완료 후 체크인
   6. 작업 결과 보고

② 긴급 사용 (장애 복구 등):
   1. 비상계정 사용 (봉인 해제)
   2. 작업 수행
   3. 1시간 내 긴급 사용 보고
   4. 24시간 내 상세 보고서 제출

제22조 (특권계정 비밀번호)
① 비밀번호 복잡도: 일반계정보다 강화 (16자 이상)
② 비밀번호 변경: 30일마다 또는 사용 후 즉시
③ 비밀번호 보관: 암호화된 금고(Vault)에 보관
④ 비밀번호 공유 금지 (개인별 계정 분리로 불필요)

--------------------------------------------------------------------------------

제8장 원격 접근

제23조 (VPN 접속)
① VPN은 외부에서 내부망 접근 시 필수
② VPN 접속 요건:
   - 2팩터 인증 (ID/PW + OTP)
   - 회사 인증서 설치
   - 보안 프로그램 설치 (백신, DLP)

③ VPN 접속 허용 시간:
   - 일반: 08:00 - 22:00
   - 야간/휴일: 사전 승인 필요

④ VPN 접속 차단:
   - 비인가 국가 IP (고위험 국가)
   - 악성코드 감염 단말
   - 보안 정책 미준수 단말

제24조 (재택근무 접근)
① 재택근무 시 접근 요건:
   - 회사 지급 단말 또는 승인된 개인 단말
   - VPN 필수 사용
   - 보안 프로그램 설치 확인

② 재택근무 시 금지 사항:
   - 공용 PC (PC방, 도서관 등) 사용
   - 공용 와이파이 사용 시 VPN 미적용
   - 화면 캡처, 자료 로컬 저장 (승인 시 제외)

제25조 (협력업체 원격 접근)
① 협력업체 원격 접근 원칙적 제한
② 불가피한 경우:
   - 계약서에 보안 조항 포함
   - 전용 VPN 계정 발급 (기간 제한)
   - 접근 범위 최소화 (해당 시스템만)
   - 실시간 모니터링

제9장 물리적 접근통제

제26조 (출입통제 구역)
| 구역 | 설명 | 통제 수준 |
|------|------|-----------|
| 일반 구역 | 사무공간 | 사원증 태그 |
| 제한 구역 | 서버실, 전산실 | 사원증 + 생체인증 |
| 통제 구역 | 금고, 백업센터 | 사원증 + 생체 + 승인 |

제27조 (출입 권한)
① 출입 권한 신청:
   - 소속 부서장 승인
   - 해당 구역 관리부서 승인
   - 정보보안팀 등록

② 출입 권한 회수:
   - 퇴직, 전배 시 즉시
   - 프로젝트 종료 시

③ 방문자 출입:
   - 사전 방문 등록
   - 안내데스크에서 방문증 수령
   - 담당자 동행 필수 (제한/통제 구역)

--------------------------------------------------------------------------------

제10장 로그 및 모니터링

제28조 (로그 기록)
① 필수 로그 항목:
   - 로그인/로그아웃 (성공/실패)
   - 권한 변경
   - 데이터 조회/수정/삭제
   - 시스템 설정 변경
   - 파일 다운로드/업로드

② 로그 보관 기간:
   - 일반 시스템: 1년
   - 금융거래시스템: 5년
   - 개인정보처리시스템: 3년

③ 로그 보호:
   - 위변조 방지 (WORM 스토리지)
   - 접근 제한 (로그 시스템 관리자만)

제29조 (모니터링)
① 실시간 모니터링:
   - 로그인 실패 5회 연속
   - 비정상 시간대 접속
   - 대량 데이터 조회/다운로드
   - 특권계정 사용

② 이상 징후 대응:
   - 자동 알림 발송
   - 보안담당자 확인
   - 필요시 계정 정지 조치
